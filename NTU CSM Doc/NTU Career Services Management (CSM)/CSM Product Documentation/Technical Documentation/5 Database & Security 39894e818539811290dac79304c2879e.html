5. Database & Security

Database

The platform uses a managed PostgreSQL 14 database on AWS RDS, configured with security and resilience in mind:

AspectHow it's handled
EncryptionEncrypted at rest
AvailabilityMulti-AZ — automatic failover across data centres
ExposureNot publicly accessible — private to the application network
AccessReachable only by the application and, for maintenance, the bastion host
Backups30-day automated backups
Deletion safetyDeletion protection enabled
CredentialsStored in AWS Secrets Manager, not in code

Data is accessed through the Drizzle ORM, over a schema of roughly 70 tables. Schema changes are shipped as versioned migration files and applied automatically — as a one-off task at the start of each deployment — never by hand against shared environments.

Authentication

  • Sign-in is handled by AWS Cognito. Each request carries a Cognito access token, which the API verifies on every request without a network round-trip. A user's sessions can be revoked instantly when needed.
  • Users have roles (admin, manager, staff, coach, student), and each part of the API checks that the caller has the right role.
  • System-to-system calls (for example, sending transactional email) use a separate, dedicated credential rather than a user login.

Two-factor authentication (2FA)

  • Two-factor authentication uses an authenticator app (TOTP) and is required for password-based accounts, which are prompted to set it up on first sign-in.
  • Users who sign in through NTU single sign-on (Azure AD) are covered by NTU's own identity controls.

Sessions

  • After sign-in, the session's refresh token is kept in a secure, server-managed cookie, and sessions can be revoked centrally.

Additional protections

  • Security headers (HSTS, Content-Security-Policy, and others) on both the API and the web app.
  • Cross-origin requests limited to approved origins.
  • Input and configuration validation on every request and at start-up.
  • Signed, verified webhooks for incoming email events.
  • Signed email-tracking links and sanitised email content.
  • Network isolation — a private database, and a QA environment restricted to the NTU network and VPN.
  • Encrypted secrets managed in AWS Secrets Manager.
