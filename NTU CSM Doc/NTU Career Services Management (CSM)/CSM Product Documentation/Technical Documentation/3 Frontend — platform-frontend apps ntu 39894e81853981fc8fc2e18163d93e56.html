3. Frontend — platform-frontend/apps/ntu

The frontend is a Next.js web application — the staff and admin interface behind csm.ntu.edu.sg. The code lives in the platform-frontend repository (the NTU app is under apps/ntu). Its development server runs on port 3100.

Structure & patterns

  • Pages are thin and delegate to containers, which hold the screen logic.
  • Application state is managed with Redux; all network calls go through a small set of API client classes (never directly from components).
  • Access to each page is controlled by route guards: a page declares whether it is protected and which permission it needs, and the app enforces this during navigation.
  • The API layer refreshes an expired sign-in token automatically and retries the request, so users aren't signed out unnecessarily.
  • Shared UI components, hooks, and utilities come from a common shared library, imported through a small set of entry points.
  • All user-facing dates and times are shown in Singapore time (Asia/Singapore).

Feature areas

  • SGCC — career-coaching sessions: overviews, session details, and CSV upload
  • Communications — email campaigns with a rich-text editor, merge fields, and recipient lists
  • Forms — public forms and submissions
  • Employer registration — public employer self-registration for CAO review
  • Users & staff, labels / segmentation, recycle bin, help, and workspace settings
  • Profile & security — including two-factor authentication setup

Authentication & security

  • Sign-in is handled by AWS Cognito, with single sign-on through NTU's identity provider (Azure AD) and two-factor authentication (authenticator app / TOTP).
  • The web app applies a Content-Security-Policy and other security headers to harden the browser experience.
