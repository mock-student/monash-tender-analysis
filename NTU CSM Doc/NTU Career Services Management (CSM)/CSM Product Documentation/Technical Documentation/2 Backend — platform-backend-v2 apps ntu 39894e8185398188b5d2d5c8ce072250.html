2. Backend — platform-backend-v2/apps/ntu

The backend is a NestJS application that powers the NTU CSM platform and serves its API at api.csm.ntu.edu.sg. The code lives in the platform-backend-v2 repository (the NTU app is under apps/ntu).

What the backend does

The API is organised into feature areas:

AI & HelpAI feedback summaries and a Notion-backed Help / FAQ

Notable technical details

  • Queue-backed email. Outgoing mail is placed on a queue and sent by a background worker, with open / click tracking, bounce handling, and suppression lists for reliable delivery.
  • Campaign engine. Campaigns resolve their audience through a targeting rule engine and can be sent once, on a recurring schedule, or triggered automatically by system events.
  • Scheduled data import. A nightly job imports student, staff, and reference data from NTU systems, decrypting the incoming files before loading them.
  • Validation everywhere. Every request and the application's own configuration are validated with Zod schemas; invalid configuration stops the service from starting.
  • Scale. Around 30 feature areas and 60+ controllers over a database of roughly 70 tables.

How the app is set up

On start-up the API applies security headers, restricts cross-origin requests to approved origins, logs requests, records an audit trail of key actions, and — in the development environment only — exposes interactive API documentation.

API documentation

  • Interactive Swagger documentation is available in the development environment.
  • A generated OpenAPI specification, request / response samples, and a ready-to-use Postman collection are kept alongside the code, together with detailed guides for the email, campaigns, templates, calendar, activity-log, and assets areas.
